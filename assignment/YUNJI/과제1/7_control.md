# 7. 피드백 제어 & PID 제어 

## 제어 시스템

제어 시스템은 어떤 대상이 원하는 상태로 움직이도록 입력을 조절하는 시스템이다. 여기서 제어 대상은 로봇, 모터, 온도 장치, 속도 장치 등이 될 수 있다.

로봇 운반차의 경우 제어 시스템은 센서가 읽은 값과 목표 상태를 비교하고, 그 차이를 줄이기 위해 모터 출력을 조절하는 역할을 한다. 예를 들어 라인트래킹 로봇은 검은색 선의 중앙을 따라가는 것이 목표이며, 센서가 로봇이 왼쪽 또는 오른쪽으로 벗어났다고 판단하면 좌우 모터 속도를 다르게 조절하여 다시 경로로 복귀한다.

### 제어의 목표

제어의 목표는 시스템이 원하는 목표값에 안정적으로 도달하고, 외부 변화가 있어도 목표 상태를 유지하도록 하는 것이다.

| 제어 목표 | 의미 | 로봇 예시 |
|---|---|---|
| 목표값 추종 | 원하는 값에 맞게 움직이는 것 | 검은색 선의 중앙을 따라 이동 |
| 안정성 | 제어 중 흔들림이나 발산이 없는 것 | 좌우로 과하게 흔들리지 않음 |
| 빠른 응답 | 변화에 빠르게 반응하는 것 | 선이 휘어질 때 빠르게 방향 수정 |
| 정확도 | 목표값과 실제값의 차이가 작은 것 | 선 중앙에서 벗어나는 정도가 작음 |
| 외란 대응 | 외부 변화에도 동작을 유지하는 것 | 바닥 마찰, 하중 변화에도 주행 유지 |

###  피드백

피드백은 시스템의 출력 결과를 다시 입력 쪽으로 되돌려 보내 제어에 활용하는 것이다. 제어 시스템은 현재 상태를 측정하고, 목표 상태와 비교한 뒤, 차이를 줄이는 방향으로 출력을 조정한다.

라인트래킹 로봇에서는 IR 센서가 현재 선의 위치를 감지한다. 이 센서값이 피드백 정보가 된다. 로봇은 센서값을 이용해 선의 중앙에서 얼마나 벗어났는지 판단하고, 좌우 모터의 속도를 조절한다.

### 목표값과 실제값

목표값은 시스템이 도달하려는 기준값이다. 실제값은 센서나 엔코더를 통해 측정된 현재 상태이다.

| 용어 | 의미 | 라인트래킹 로봇 예시 |
|---|---|---|
| 목표값(Setpoint) | 원하는 기준값 | 검은색 선의 중앙 위치 |
| 실제값(Process Variable) | 현재 측정된 값 | IR 센서가 측정한 현재 선 위치 |
| 제어 입력(Control Input) | 제어기가 내보내는 명령 | 좌우 모터 PWM 값 |
| 출력(Output) | 실제 시스템의 결과 | 로봇의 이동 방향과 속도 |

### 오차

오차는 목표값과 실제값의 차이를 의미한다.

```text
오차 = 목표값 - 실제값
```

라인트래킹 로봇에서 오차가 0에 가까우면 로봇이 선의 중앙을 잘 따라가고 있다는 뜻이다. 오차가 양수 또는 음수로 커지면 로봇이 선의 한쪽으로 벗어났다는 뜻이며, 제어기는 이 오차를 줄이도록 모터 출력을 조정한다.

---

## 피드백 제어의 기본 구조

**제어 시스템의 구성 요소**

피드백 제어 시스템은 보통 다음과 같은 구성 요소로 이루어진다.

| 구성 요소 | 역할 | 로봇 예시 |
|---|---|---|
| 목표값 | 시스템이 따라가야 하는 기준 | 선의 중앙, 목표 속도 |
| 비교기 | 목표값과 실제값을 비교 | 오차 계산 |
| 제어기 | 오차를 바탕으로 제어 명령 생성 | PID 제어 알고리즘 |
| 구동기 | 제어 명령을 실제 힘이나 움직임으로 변환 | 모터 드라이버, DC 모터 |
| 제어 대상 | 제어되는 실제 시스템 | 로봇 본체와 바퀴 |
| 센서 | 실제 상태를 측정 | IR 센서, 엔코더 |
| 피드백 경로 | 측정값을 제어기로 되돌림 | 센서값 입력 |

```text
목표값 → 비교기 → 제어기 → 모터 드라이버/모터 → 로봇 이동
          ↑                                      ↓
          └──────────── 센서 피드백 ────────────┘
```

**오픈 루프 제어**

오픈 루프 제어는 출력 결과를 다시 확인하지 않고 미리 정한 입력만 주는 방식이다. 예를 들어 모터에 항상 PWM 100을 주면 로봇이 일정하게 움직일 것이라고 가정하는 방식이다.

오픈 루프 제어는 구조가 단순하지만, 바닥 마찰, 배터리 전압, 적재물 무게, 모터 편차가 바뀌면 실제 속도나 방향이 달라질 수 있다. 즉, 결과를 확인하지 않기 때문에 오차를 자동으로 보정하기 어렵다.

**클로즈드 루프 제어**

클로즈드 루프 제어는 센서로 실제 출력 결과를 측정하고, 목표값과 비교하여 제어 입력을 계속 수정하는 방식이다. 피드백 제어라고도 부른다.

로봇 운반차에서는 IR 센서로 선의 위치를 확인하고, 엔코더로 바퀴 속도나 이동 거리를 확인할 수 있다. 이 값을 이용하면 로봇이 경로에서 벗어났을 때 자동으로 방향을 수정하거나, 목표 속도와 실제 속도 차이를 줄일 수 있다.

| 구분 | 오픈 루프 제어 | 클로즈드 루프 제어 |
|---|---|---|
| 피드백 사용 | 사용하지 않음 | 사용함 |
| 구조 | 단순함 | 비교적 복잡함 |
| 외란 대응 | 약함 | 강함 |
| 정확도 | 낮을 수 있음 | 높음 |
| 예시 | 일정 PWM으로 모터 구동 | 센서값을 보고 모터 속도 조절 |

## PID 제어의 개념

PID 제어는 오차를 줄이기 위해 P, I, D 세 가지 항을 함께 사용하는 대표적인 피드백 제어 방법이다. PID는 Proportional, Integral, Derivative의 약자이다.

```text
PID 제어 = P 제어 + I 제어 + D 제어
```

PID 제어기는 목표값과 실제값의 차이인 오차를 입력으로 받아 제어 출력을 계산한다. 로봇에서는 이 제어 출력이 좌우 모터 속도 차이나 PWM 값으로 사용될 수 있다.

라인트래킹 로봇에서는 선의 중앙에서 벗어난 정도를 오차로 두고 PID 제어를 적용할 수 있다. 오차가 크면 더 강하게 방향을 틀고, 오차가 누적되면 계속 한쪽으로 치우치는 문제를 보정하며, 오차가 급격히 변하면 흔들림을 줄이는 방향으로 제어할 수 있다.

## P, I, D 각 항목의 역할과 수학적 표현

### 전체 PID 제어식

연속 시간에서 PID 제어식은 다음과 같이 표현할 수 있다.

```text
u(t) = Kp·e(t) + Ki·∫e(t)dt + Kd·de(t)/dt
```

| 기호 | 의미 |
|---|---|
| u(t) | 제어 출력 |
| e(t) | 시간 t에서의 오차 |
| Kp | 비례 이득 |
| Ki | 적분 이득 |
| Kd | 미분 이득 |

마이크로컨트롤러에서는 보통 일정한 시간 간격으로 센서값을 읽기 때문에 이산 형태로 구현한다.

```text
P = Kp × 현재 오차
I = I + Ki × 현재 오차 × Δt
D = Kd × (현재 오차 - 이전 오차) / Δt
출력 = P + I + D
```

여기서 Δt는 제어 주기이다.

---

###  P 제어

P 제어는 **현재 오차에 비례하여 제어 출력을 만드는 항**이다.

```text
P = Kp × e(t)
```

오차가 크면 큰 제어 출력을 내고, 오차가 작으면 작은 제어 출력을 낸다. 라인트래킹 로봇에서는 선에서 많이 벗어날수록 좌우 모터 속도 차이를 크게 만들어 빠르게 복귀하도록 한다.

#### P 항의 특징

- 반응이 빠르다.
- 구현이 쉽다.
- 오차가 클수록 강하게 제어한다.
- Kp가 너무 작으면 반응이 느리다.
- Kp가 너무 크면 로봇이 좌우로 심하게 흔들릴 수 있다.
- P 제어만으로는 일정한 잔류 오차가 남을 수 있다.

---

###  I 제어

I 제어는 **오차가 시간에 따라 누적된 값**을 이용하는 항이다.

```text
I = Ki × ∫e(t)dt
```

I 항은 오차가 계속 같은 방향으로 남아 있을 때 그 오차를 점점 크게 반영한다. 따라서 P 제어만으로 남는 정상 상태 오차를 줄이는 데 도움이 된다.

라인트래킹 로봇에서 로봇이 계속 선의 한쪽으로 치우치는 경우, I 항은 누적된 오차를 이용해 보정량을 증가시켜 중앙으로 돌아오게 할 수 있다.

#### I 항의 특징

- 잔류 오차를 줄이는 데 효과적이다.
- 작은 오차가 오래 지속될 때 보정할 수 있다.
- Ki가 너무 크면 오버슈트가 커질 수 있다.
- 출력 제한이 있는 상황에서 적분값이 계속 쌓이는 적분 포화 현상이 생길 수 있다.
- 빠른 라인트래킹에서는 I 항을 너무 크게 쓰면 반응이 둔해질 수 있다.

---

###  D 제어

D 제어는 **오차가 얼마나 빠르게 변하는지**를 이용하는 항이다.

```text
D = Kd × de(t)/dt
```

마이크로컨트롤러에서는 다음처럼 근사한다.

```text
D = Kd × (현재 오차 - 이전 오차) / Δt
```

D 항은 오차의 변화 속도를 보고 앞으로 오차가 커질지 줄어들지를 예측하는 역할을 한다. 라인트래킹 로봇에서는 급격한 방향 변화나 좌우 흔들림을 줄이는 데 도움을 줄 수 있다.

#### D 항의 특징

- 빠른 변화에 대응한다.
- 오버슈트와 진동을 줄이는 데 도움이 된다.
- 시스템을 더 안정적으로 만들 수 있다.
- 센서 노이즈에 민감하다.
- D 항을 사용할 때는 센서값 필터링이 필요할 수 있다.

## PID 제어가 로봇에 적용되는 방식

### 라인트래킹 제어

라인트래킹 로봇에서는 IR 센서 배열을 이용해 검은색 선의 위치를 계산한다. 선이 로봇의 중앙에 있으면 오차는 0이고, 선이 왼쪽이나 오른쪽으로 치우치면 오차가 발생한다.

```text
목표값: 선이 센서 배열의 중앙에 위치
실제값: 현재 감지된 선의 위치
오차: 목표 위치 - 현재 선 위치
PID 출력: 좌우 모터 속도 차이
```

PID 출력값을 이용해 좌우 모터 속도를 조절하면 다음과 같이 제어할 수 있다.

```text
기본 속도 = baseSpeed
보정값 = PID 출력

왼쪽 모터 속도 = baseSpeed - 보정값
오른쪽 모터 속도 = baseSpeed + 보정값
```

이때 보정값이 양수인지 음수인지에 따라 로봇은 왼쪽 또는 오른쪽으로 방향을 바꾼다.

### 엔코더 속도 제어 예시

엔코더 모터를 사용할 경우 목표 속도와 실제 속도를 비교하여 모터 PWM을 조절할 수 있다.

```text
오차 = 목표 속도 - 실제 속도
PID 출력 = PWM 보정값
```

이 방식은 배터리 전압이 낮아지거나 적재물이 무거워져도 목표 속도를 유지하는 데 도움이 된다.

## PID 튜닝

PID 튜닝은 Kp, Ki, Kd 값을 조절하여 시스템이 원하는 성능을 내도록 맞추는 과정이다. 튜닝이 잘못되면 로봇이 너무 느리게 반응하거나, 좌우로 심하게 흔들리거나, 목표값을 지나쳐 오버슈트가 커질 수 있다.

### 각 이득값이 커질 때의 일반적인 영향

| 항목 | 값을 키우면 생기는 효과 | 너무 크면 생기는 문제 |
|---|---|---|
| Kp | 반응이 빨라짐 | 진동, 오버슈트 증가 |
| Ki | 잔류 오차 감소 | 적분 포화, 느린 진동 발생 |
| Kd | 오버슈트와 흔들림 감소 | 노이즈에 민감, 출력이 불안정해질 수 있음 |

**수동 튜닝 방법**

로봇 프로젝트에서는 간단한 수동 튜닝 방법을 많이 사용한다.

1. Ki와 Kd를 0으로 둔다.
2. Kp를 조금씩 증가시킨다.
3. 로봇이 선을 따라가기 시작하지만 심하게 흔들리지 않는 Kp 값을 찾는다.
4. 선을 따라가지만 한쪽으로 치우치는 오차가 남으면 Ki를 아주 조금 추가한다.
5. 곡선 구간에서 오버슈트나 흔들림이 크면 Kd를 조금 추가한다.
6. 실제 주행 환경에서 직선, 곡선, 교차 구간을 테스트하며 값을 조정한다.

라인트래킹 로봇에서는 I 항을 너무 크게 넣으면 누적 오차 때문에 오히려 경로를 늦게 따라가거나 크게 흔들릴 수 있다. 따라서 처음에는 P 또는 PD 제어부터 적용하고, 필요할 때만 I 항을 작게 추가하는 방식이 적절하다.

**Ziegler-Nichols 튜닝 개요**

Ziegler-Nichols 방법은 고전적인 PID 튜닝 방법 중 하나이다. 일반적으로 I와 D 항을 끄고 P 이득을 올려 시스템이 지속적으로 진동하는 지점을 찾은 뒤, 그때의 이득과 진동 주기를 이용해 PID 값을 계산한다.

이 방법은 PID 값을 빠르게 얻는 데 도움이 되지만, 실제 로봇에서는 진동을 의도적으로 발생시켜야 하므로 안전에 주의해야 한다. 특히 운반 로봇처럼 하중이 있거나 바닥에서 움직이는 장치에서는 낮은 속도에서 테스트하고, 충돌을 방지한 상태에서 적용해야 한다.

### 튜닝 시 주의할 점

- 한 번에 하나의 값을 조정한다.
- 배터리 전압이 충분한 상태에서 테스트한다.
- 직선과 곡선 구간을 모두 테스트한다.
- 센서값이 흔들리면 먼저 센서 필터링을 적용한다.
