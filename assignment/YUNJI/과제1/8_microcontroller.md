# 8. 마이크로컨트롤러

## 마이크로컨트롤러

마이크로컨트롤러(Microcontroller, MCU)는 하나의 칩 안에 CPU, 메모리, 입출력 장치, 타이머, 통신 장치 등이 포함된 작은 컴퓨터이다. 일반 컴퓨터가 여러 작업을 범용적으로 처리하는 데 초점을 둔다면, 마이크로컨트롤러는 센서 읽기, 모터 제어, LED 제어, 통신 처리처럼 특정 장치를 제어하는 데 사용된다.

로봇에서 마이크로컨트롤러는 사람의 두뇌와 비슷한 역할을 한다. 센서에서 들어오는 값을 읽고, 현재 상태를 판단한 뒤, 모터 드라이버나 서보 모터에 제어 신호를 보낸다.

본 프로젝트의 운반 로봇은 하얀색 바닥에 표시된 검은색 운송 경로를 따라 이동해야 하므로, 마이크로컨트롤러는 IR 센서의 값을 읽어 로봇이 선의 왼쪽 또는 오른쪽으로 벗어났는지 판단하고, 좌우 바퀴의 속도를 조절해야 한다.

## 마이크로컨트롤러의 역할

| 역할 | 설명 |
|---|---|
| 센서 입력 처리 | IR 센서, 초음파 센서, 엔코더 등에서 들어오는 값을 읽는다. |
| 판단 및 제어 연산 | 센서값을 바탕으로 로봇의 현재 상태를 판단하고 제어 출력을 계산한다. |
| 모터 제어 | PWM 신호와 방향 제어 신호를 이용해 DC 모터의 속도와 방향을 제어한다. |
| 서보 제어 | PWM 신호를 이용해 서보 모터의 각도를 제어한다. |
| 통신 처리 | PC, 다른 보드, 센서 모듈, 디스플레이 등과 데이터를 주고받는다. |
| 시간 관리 | 타이머를 이용해 일정 주기로 센서값을 읽거나 제어 루프를 실행한다. |
| 데이터 기록 | 이동 시간, 센서값, 엔코더 카운트, 속도 등을 저장하거나 외부 장치로 전송한다. |

즉, 마이크로컨트롤러는 로봇의 입력, 처리, 출력 과정을 연결하는 중심 장치이다.

```text
센서 입력 → 마이크로컨트롤러 판단 → 모터 출력 → 로봇 이동
```

## 기본 제어 방식과 인터페이스

마이크로컨트롤러는 여러 종류의 핀과 제어 신호를 이용해 외부 부품을 제어한다. 로봇 제어에서 자주 사용하는 인터페이스에는 디지털 핀, 아날로그 핀, PWM 출력이 있다.

### 디지털 핀

디지털 핀은 HIGH 또는 LOW처럼 두 가지 상태를 다루는 핀이다. 일반적으로 HIGH는 전압이 높은 상태, LOW는 전압이 낮은 상태를 의미한다.

디지털 핀은 다음과 같은 용도로 사용된다.

- LED 켜기/끄기
- 버튼 입력 읽기
- 모터 드라이버의 방향 신호 제어
- 디지털 센서값 읽기
- 릴레이, 부저, 스위치 제어

예를 들어 DC 모터 드라이버의 IN1, IN2 핀에 디지털 신호를 보내면 모터의 회전 방향을 정할 수 있다.

```text
IN1 = HIGH, IN2 = LOW  → 정방향 회전
IN1 = LOW,  IN2 = HIGH → 역방향 회전
IN1 = LOW,  IN2 = LOW  → 정지
```

### 아날로그 핀

아날로그 핀은 연속적인 전압 값을 읽는 데 사용된다. 마이크로컨트롤러는 아날로그 값을 그대로 처리하기 어렵기 때문에 ADC(Analog-to-Digital Converter)를 이용해 전압을 숫자로 변환한다.

예를 들어 10비트 ADC는 입력 전압을 0부터 1023까지 총 1024단계로 변환할 수 있다.

아날로그 핀은 다음과 같은 용도로 사용된다.

- IR 센서의 반사율 값 읽기
- 가변저항 값 읽기
- 조도 센서 값 읽기
- 온도 센서 값 읽기
- 배터리 전압 측정

라인트래킹 로봇에서는 IR 센서의 아날로그 값을 읽어 검은색 선과 흰색 바닥의 반사율 차이를 판단할 수 있다.

### PWM

PWM(Pulse Width Modulation)은 디지털 신호를 빠르게 켰다 껐다 하면서 평균적으로 전달되는 전력을 조절하는 방식이다. PWM은 DC 모터의 속도 제어와 서보 모터 제어에 많이 사용된다.

PWM에서 중요한 개념은 듀티비(Duty Cycle)이다. 듀티비는 한 주기에서 HIGH 상태가 유지되는 비율을 의미한다.

```text
듀티비 0%   → 항상 꺼짐
듀티비 50%  → 절반 시간만 켜짐
듀티비 100% → 항상 켜짐
```

DC 모터에서는 듀티비가 커질수록 모터에 전달되는 평균 전력이 커져 회전 속도가 증가한다. 따라서 마이크로컨트롤러는 PWM 출력으로 모터 드라이버를 제어하여 좌우 바퀴의 속도를 조절할 수 있다.

라인트래킹 로봇에서는 검은색 선이 왼쪽으로 감지되면 오른쪽 바퀴와 왼쪽 바퀴의 PWM 값을 다르게 주어 방향을 수정할 수 있다.

## 마이크로컨트롤러의 통신 방식

로봇은 센서, 모터 드라이버, 디스플레이, 다른 제어 보드, 컴퓨터 등과 데이터를 주고받아야 한다. 이를 위해 마이크로컨트롤러는 UART, I2C, SPI 같은 통신 방식을 사용한다.

### UART

UART(Universal Asynchronous Receiver/Transmitter)는 비동기 직렬 통신 방식이다. 일반적으로 TX와 RX 두 개의 신호선을 사용한다.

| 구분 | 설명 |
|---|---|
| TX | 데이터를 보내는 선 |
| RX | 데이터를 받는 선 |
| 특징 | 클록선 없이 송신부와 수신부가 같은 통신 속도(baud rate)를 맞춰 통신 |

UART는 다음과 같은 용도로 사용된다.

- PC와 마이크로컨트롤러 간 시리얼 모니터 통신
- 블루투스 모듈과 통신
- GPS 모듈과 통신
- 디버깅 메시지 출력

로봇 개발 중에는 UART를 이용해 센서값, 모터 출력값, 엔코더 카운트 등을 PC로 출력하여 동작을 확인할 수 있다.

###  I2C

I2C(Inter-Integrated Circuit)는 여러 장치를 두 개의 선으로 연결할 수 있는 통신 방식이다. 일반적으로 SDA와 SCL 두 개의 선을 사용한다.

| 구분 | 설명 |
|---|---|
| SDA | 데이터 선 |
| SCL | 클록 선 |
| 특징 | 하나의 버스에 여러 장치를 주소로 구분하여 연결 가능 |

I2C는 다음과 같은 장치에 많이 사용된다.

- OLED 디스플레이
- IMU 센서
- 온습도 센서
- 거리 센서
- RTC 모듈

I2C는 배선이 적고 여러 장치를 연결하기 쉬워 소형 로봇에서 유용하다.

### SPI

SPI(Serial Peripheral Interface)는 빠른 데이터 전송이 가능한 동기식 직렬 통신 방식이다. 일반적으로 SCK, MOSI, MISO, CS 선을 사용한다.

| 구분 | 설명 |
|---|---|
| SCK | 클록 선 |
| MOSI | 마스터에서 슬레이브로 데이터 전송 |
| MISO | 슬레이브에서 마스터로 데이터 전송 |
| CS | 통신할 장치를 선택하는 선 |

SPI는 I2C보다 배선이 많지만 빠른 통신이 가능하다. SD 카드 모듈, 고속 센서, 디스플레이, 무선 통신 모듈 등에 사용된다.

본 프로젝트에서 주행 데이터를 저장하기 위해 SD 카드 모듈을 사용할 경우 SPI 통신을 사용할 수 있다.

### 통신 방식 비교

| 통신 방식 | 사용 선 | 특징 | 활용 예시 |
|---|---|---|---|
| UART | TX, RX | 구조가 단순하고 디버깅에 유리 | PC 통신, 블루투스, GPS |
| I2C | SDA, SCL | 적은 선으로 여러 장치 연결 가능 | 센서, OLED, RTC |
| SPI | SCK, MOSI, MISO, CS | 빠른 데이터 전송 가능 | SD 카드, 디스플레이, 고속 센서 |

## 실시간 제어와 타이머

### 시간 제어의 개념

실시간 제어는 정해진 시간 안에 센서값을 읽고, 제어 연산을 수행하고, 출력 신호를 갱신하는 제어 방식이다. 로봇은 계속 움직이기 때문에 센서값을 한 번만 읽고 끝내는 것이 아니라 일정한 주기로 반복해서 읽어야 한다.

라인트래킹 로봇의 경우 다음 과정이 매우 빠르게 반복된다.

```text
IR 센서값 읽기
→ 선 위치 판단
→ 오차 계산
→ 좌우 모터 PWM 갱신
→ 다시 센서값 읽기
```

이 과정이 너무 느리면 로봇이 검은색 선을 벗어난 뒤에야 방향을 수정하게 된다. 따라서 일정한 주기로 제어 루프를 실행하는 것이 중요하다.

### 타이머의 개념

타이머는 마이크로컨트롤러 내부에서 시간을 측정하거나 일정 시간마다 이벤트를 발생시키는 장치이다. 타이머는 클록 신호를 세면서 동작하며, 설정한 값에 도달하면 인터럽트를 발생시키거나 PWM 신호를 만들 수 있다.

타이머는 다음과 같은 용도로 사용된다.

- 일정 주기로 센서값 읽기
- 일정 주기로 PID 제어 계산 실행
- PWM 신호 생성
- 엔코더 펄스 측정
- 시간 지연 구현
- 주행 시간 측정

### 타이머의 사용 절차

타이머를 사용해 주기적으로 제어를 수행하려면 일반적으로 다음 절차를 따른다.

1. 제어 주기를 정한다.
   - 예: 10ms마다 센서값 읽기
   - 예: 20ms마다 PID 제어 계산

2. 타이머 클록과 분주비를 설정한다.
   - 마이크로컨트롤러의 기본 클록을 타이머에 맞게 나누어 사용한다.

3. 타이머 비교값 또는 오버플로 조건을 설정한다.
   - 설정한 시간마다 이벤트가 발생하도록 한다.

4. 인터럽트 사용 여부를 정한다.
   - 인터럽트를 사용하면 정해진 시간마다 자동으로 특정 함수가 실행된다.

5. 인터럽트 서비스 루틴 또는 제어 루프를 작성한다.
   - 센서값 읽기, 오차 계산, PWM 갱신 등을 수행한다.

6. 실제 동작을 확인하고 주기를 조정한다.
   - 로봇 속도와 센서 반응에 맞춰 제어 주기를 조정한다.

### 타이머 사용 예시

라인트래킹 로봇에서 10ms마다 제어 루프를 실행한다고 가정하면 다음과 같은 구조가 된다.

```text
10ms마다 실행:
1. IR 센서값 읽기
2. 검은색 선 위치 계산
3. 목표 위치와 실제 위치의 오차 계산
4. PID 또는 비례 제어로 보정값 계산
5. 왼쪽/오른쪽 모터 PWM 갱신
```

이렇게 타이머를 사용하면 로봇의 제어 주기가 일정해져 주행이 더 안정적이 된다.

## 아두이노

### 마이크로보드의 개념

마이크로보드는 마이크로컨트롤러 칩을 사용하기 쉽도록 전원 회로, USB 연결부, 입출력 핀, 통신 핀 등을 보드 형태로 만든 것이다. 마이크로컨트롤러 칩만 사용하려면 전원, 클록, 업로드 회로 등을 직접 구성해야 하지만, 마이크로보드는 이러한 회로가 기본적으로 포함되어 있어 개발과 실습이 쉽다.

마이크로보드의 예시는 다음과 같다.

- Arduino UNO
- Arduino Nano
- Raspberry Pi Pico
- ESP32 개발 보드
- STM32 Nucleo 보드

### 아두이노란?

아두이노(Arduino)는 마이크로컨트롤러 보드와 개발 환경을 포함하는 오픈소스 전자 프로토타이핑 플랫폼이다. 사용자는 아두이노 IDE에서 코드를 작성하고, USB 케이블로 보드에 업로드하여 센서와 모터 같은 외부 장치를 제어할 수 있다.

아두이노는 초보자가 사용하기 쉽고 예제와 라이브러리가 많아 로봇 제어 실습에 적합하다. 라인트래킹 로봇을 만들 때도 IR 센서값 읽기, 모터 PWM 제어, 시리얼 출력, 서보 제어 등을 비교적 쉽게 구현할 수 있다.

### 아두이노의 기본 구조

아두이노 프로그램은 보통 `setup()` 함수와 `loop()` 함수로 구성된다.

```cpp
void setup() {
  // 처음 한 번 실행되는 코드
}

void loop() {
  // 계속 반복 실행되는 코드
}
```

`setup()`에서는 핀 모드 설정, 시리얼 통신 시작, 센서 초기화 등을 수행한다. `loop()`에서는 센서값을 계속 읽고, 로봇의 상태를 판단하고, 모터를 제어한다.

### 아두이노의 인터페이스

아두이노에서 자주 사용하는 인터페이스는 다음과 같다.

| 인터페이스 | 설명 | 로봇 활용 예시 |
|---|---|---|
| 디지털 입력 | HIGH/LOW 값을 읽음 | 버튼, 디지털 IR 센서 |
| 디지털 출력 | HIGH/LOW 값을 출력 | LED, 모터 방향 제어 |
| 아날로그 입력 | 연속적인 전압 값을 읽음 | 아날로그 IR 센서, 배터리 전압 측정 |
| PWM 출력 | 듀티비를 조절한 신호 출력 | DC 모터 속도 제어, 서보 제어 |
| UART | 직렬 통신 | PC 디버깅, 블루투스 |
| I2C | 2선식 통신 | OLED, IMU 센서 |
| SPI | 고속 직렬 통신 | SD 카드, 디스플레이 |

### 아두이노를 이용한 로봇 제어 흐름

아두이노를 이용해 라인트래킹 운반 로봇을 제어하면 다음과 같은 흐름이 된다.

```text
1. IR 센서가 검은색 선과 흰색 바닥의 반사율 차이를 감지한다.
2. 아두이노가 센서값을 아날로그 또는 디지털 핀으로 읽는다.
3. 아두이노가 선의 위치를 판단한다.
4. 선에서 벗어난 정도에 따라 좌우 모터 PWM 값을 계산한다.
5. 모터 드라이버에 방향 신호와 PWM 신호를 보낸다.
6. DC 기어드 모터가 바퀴를 회전시킨다.
7. 엔코더가 있다면 바퀴 회전량과 속도를 다시 아두이노로 전달한다.
8. 아두이노가 이 정보를 바탕으로 계속 제어를 반복한다.
```
